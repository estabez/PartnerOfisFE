1                        General

The purpose of this document is to verify the correctness of the implemented different E2E topology views and Connectivity Matrix custom GUI. This document will address the test case scenarios which can be used for the needed validations.

1.1   Scope

The document will address test case scenarios which will be used to validate and test the different E2E topology views and Connectivity Matrix custom GUI for import, export, searching and updating/deleting/adding Access Connectivity objects. Validate and test all of them against the approved LLD for E2E topology views.

 

1.2                                   Rules & Constraints

 

  To conduct the TCs execution, all stakeholders should have the latest version of the LLD.

  Each of the Test cases (TC) should be verified against the approved latest version of LLD.

  The Test cases (TC) would verify the population of the values in mandatory and optional attributes for each object instance.

  The Optional attributes population for all the instances depends upon their values in imported data files or in custom matrix GUI.

  The ATP would focus on positive & negative aspects of TCs, where in the TCs will focus on the validity of data to meet the requirements & specifications described in approved LLD for each data inserted.

  To accomplish the ATP, the TCs would be verified within the domain of attributes types as specified in CDRM & DSDs
